# 🚀 [260311] 마음 약방: 추가 작업 계획서 (Phase 4 ~ 6)

## Phase 4: 임시 데이터 파이프라인 구축 (Local Mock Data) [완료 ✅]

백엔드 연동 전 UI/UX와 비즈니스 로직이 매끄럽게 돌아가는지 확인하기 위해 data.json을 활용한 임시 데이터 레이어를 구축했습니다.

### 4.1. Remote Data Source Mocking [완료 ✅]

- 실제 Gemini API 호출을 보류하고, Future.delayed를 사용하여 네트워크 지연을 모방합니다 (2초 대기 후 로딩 UI 표시).
- 선택한 스타일(공감형/이성형/온기형)에 따라 미리 준비된 따뜻한 위로 문구 세트를 반환하도록 더미 데이터를 구성했습니다.

### 4.2. Local Data Source 파일 입출력 [완료 ✅]

- Hive 대신 path_provider와 dart:io를 활용하여 디바이스의 Documents 디렉토리에 data.json 파일로 임시 저장합니다.
- 처방전 조제 완료 시 모델을 JSON 직렬화하여 누적 저장하고, 앱을 재시작해도 '내 처방전' 탭에서 이전에 조제받은 기록을 성공적으로 불러올 수 있습니다.

## Phase 5: 리텐션 및 디테일 고도화 (Retention & Polish) [완료 ✅]

사용자가 앱에 지속적으로 방문하도록 돕는 감성적인 디테일 요소들을 추가했습니다.

### 5.1. 나만의 약장 (마음 일기 그리드 뷰) [완료 ✅]

- 내 처방전 리스트를 최신순으로 정렬하고 부드러운 스크롤 애니메이션(Staggered Fade-In/Slide)을 적용했습니다.
- 개별 처방전을 눌렀을 때 나타나는 다이얼로그 모달을 BounceTappable 햅틱과 연결하여 찰진 촉각 경험을 완성했습니다.

### 5.2. 감정 분석 차트 [완료 ✅]

- **데이터 시각화**: l_chart 패키지를 도입하여 '나의 마음 처방 통계' 도넛(Pie) 차트를 구현했습니다.
- 사용자가 지금까지 어떤 유형(공감형/이성형/온기형)의 위로를 가장 많이 찾았는지 한눈에 파악할 수 있도록 '내 처방전' 페이지 상단에 통합했습니다. 빈 값이 0%인 경우 섹션을 자연스럽게 숨겨 렌더링 오류를 막았습니다.

### 5.3. 성능 및 접근성 최적화 [완료 ✅]

- AnimatedBuilder 내장, const SizedBox 활용 등 불필요한 위젯 트리 리렌더링을 방지했습니다.

## Phase 6: 실제 백엔드 연동 (Real Data Connection) [다음 진행 단계 ⏳]

임시 data.json을 실제 클라우드 인프라와 연결합니다.

### 6.1. Supabase 백엔드 연동

- 익명 로그인 또는 소셜 로그인을 통해 유저 식별 로직을 추가합니다.
- prescriptions 테이블 연결 및 RLS(Row Level Security) 정책을 구성하여 보안을 확립합니다.

### 6.2. Hive를 통한 오프라인-First 도입

- data.json 대신 구조화되고 빠른 Hive 로컬 DB를 연동하여 성능을 높이고 Supabase와 백그라운드 동기화를 처리합니다.

### 6.3. 실제 AI 생성 파이프라인 연동

- 준비된 API 키를 입력하여 Gemini/Claude 등 LLM의 실제 프롬프트 응답 결과를 파싱합니다.
