# 🧠 마음 약방: 지능형 조제 로직 가이드 (Gemini Edition)

사용자의 감정을 분석하고, 가장 적절한 위로의 문장으로 '마음의 약'을 조제하는 AI 엔진 가이드입니다.

## 🚧 현재 구현 상태: Phase 4 (Mock API)

현재는 앱 내 AIRemoteDataSource에서 Future.delayed(2s)를 이용한 **가짜 지연 시간(Mocking)**과, 선택한 스타일에 따른 하드코딩된 더미 텍스트를 반환하여 프론트엔드 UI/UX 흐름을 완벽히 테스트하고 있습니다. 추후 Phase 6에서 아래의 실제 AI 연동 로직이 부착될 예정입니다.

---

## 1. 예정된 AI 모델 사양 (Model Specification)

- **Engine**: Google Gemini 1.5 Flash
- **Response Mode**: Strict JSON Output
- **Prompt Strategy**: Zero-shot with Persona Injection

## 2. 3대 조제 스타일 (Prescription Styles)

사용자는 고민 접수 시 자신의 현재 상태에 맞는 약사의 페르소나를 선택할 수 있습니다.

| 스타일 코드 | 명칭 (Name) | 페르소나 설명 (Persona)                       | 주요 톤앤매너                                            |
| :---------- | :---------- | :-------------------------------------------- | :------------------------------------------------------- |
| **F**       | **공감형**  | 마음을 어루만지는 따뜻한 위로와 정서적 지지   | "많이 힘들었죠?", "그 마음 제가 알아요."                 |
| **T**       | **이성형**  | 냉철하지만 현실적인 해결책과 객관적 상황 분석 | "원인은 이것일 수 있어요.", "이렇게 해보는 건 어떨까요?" |
| **W**       | **온기형**  | 포근한 격려와 아날로그적인 감성, 시적인 표현  | "밤하늘의 별처럼...", "따뜻한 차 한 잔의 여유를."        |

## 3. 프롬프트 구조 (Prompt Architecture)

AIRemoteDataSource의 \_getSystemPrompt 메서드에서 관리되는 시스템 프롬프트의 핵심 구성 요소는 다음과 같습니다.

### A. 역할 정의 (Role)

"당신은 심야의 AI 약사입니다. 사용자의 고민을 듣고, 마음을 어루만지는 처방전 제목, 상세 처방 내용, 그리고 위로가 되는 한 줄 명언(quote)을 작성해 주세요."

### B. 창의적 제약 (Constraints)

- **은유적 제목**: "반짝이는 고독의 연고", "젖은 마음을 말리는 볕" 등 아날로그 감성이 느껴지는 창의적인 제목 사용.
- **JSON 형식**: 반드시 지정된 스키마({title, content, quote})를 준수하여 파싱 오류 방지.
- **구어체**: 기계적인 말투를 배제하고, 사람이 직접 말해주는 듯한 다정한 말투 사용.

## 4. 데이터 스키마 (Response Schema)

`json
{
  "title": "은유적인 처방전 제목 (string)",
  "content": "선택된 스타일에 맞춘 상세 위로 내용 (string)",
  "quote": "가슴에 남는 한 줄 명언 (string)"
}
`

## 5. 예외 처리 로직 (Safety & Fallback)

- **API 장애 시**: AI 서버 응답이 없을 경우, "잠시 쉬어가는 시간"이라는 기본 처방전과 함께 "잠시 멈추어도 괜찮습니다"라는 위로 문구를 반환하여 사용자가 앱 오류로 느끼지 않도록 처리합니다.
- **부적절한 입력**: Gemini의 Safety Filter 기능을 활용하여 공격적이거나 부적절한 입력에 대해서는 정중하게 답변을 거절하도록 설정할 예정입니다.
