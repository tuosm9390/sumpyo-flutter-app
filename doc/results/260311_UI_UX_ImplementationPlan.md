# 🎨 [260311] 마음 약방 (구 숨표 AI): 따뜻한 디지털 약방 UI/UX 구현 로드맵 (개편안)

본 문서는 '마음의 쉼표'라는 서비스 철학을 시각적, 촉각적으로 완벽하게 구현하기 위한 UI/UX 로드맵입니다. 최신 UI 트렌드(부드러운 모션, 반응형 상호작용)를 차용하되, 차갑고 기계적인 느낌을 배제하고 **아날로그적인 따뜻함과 종이 질감**을 메인 컨셉으로 삼습니다.

## 1. 디자인 철학 및 무드 (Design Philosophy)

- **현대적 아날로그 (Modern Analog)**: 눈이 편안한 웜 화이트와 종이 질감(Paper Texture)을 바탕으로, 디지털의 차가움을 중화시키는 따뜻한 분위기를 연출합니다.
- **마이크로 인터랙션 (Micro-interactions)**: 60fps/120fps 네이티브 성능을 극대화하여, 생동감 있는 피드백(바운스, 플로팅, 햅틱)으로 감성적 연결고리를 형성합니다.
- **선택적 집중 (Selective Focus)**: 여백을 충분히 두고, 심리적 압박감을 주지 않는 부드러운 UI 요소(부드러운 그림자, 둥근 모서리)를 배치합니다.

## 2. 시각적 가이드라인 (Visual Guidelines) [완료 ✅]

### 🎨 컬러 팔레트 (App Color Palette) [완료 ✅]

시력 보호와 심리적 안정을 위해 너무 쨍한 원색이나 순백색(#FFFFFF)은 피하고, 자연의 색을 담습니다.

| 이름              | 색상 코드 | 용도                                    | 감성 효과            |
| ----------------- | --------- | --------------------------------------- | -------------------- |
| **Warm White**    | #F9F8F6   | 전체 배경, 종이 느낌의 카드             | 편안함, 깨끗함       |
| **Sage Green**    | #B7C9B0   | 주 포인트 (약사 가운, 주요 버튼, 강조)  | 안정감, 치유, 자연   |
| **Mute Blue**     | #A1B5C1   | 보조 포인트 (음악/청각 요소, 헤더 배경) | 차분함, 신뢰, 밤하늘 |
| **Soft Charcoal** | #4A4A4A   | 주요 텍스트 (가독성 고려)               | 차분함, 고급스러움   |

### 🔠 타이포그래피 (Typography) [완료 ✅]

- **앱 UI (기본/본문)**: Pretendard - 모바일에 최적화된 높은 가독성과 깔끔함.
- **감성 텍스트 (처방전 내용)**: 고운바탕(Gowun Batang) - 따뜻한 아날로그 감성을 전달.
- **강조 숫자 (처방전 번호, 포인트)**: Montserrat - 세련되고 현대적인 감각 추가.

## 3. 모바일 특화 인터랙션 (Flutter Animation Principles) [완료 ✅]

모든 움직임은 60fps의 부드러움을 유지하며 물리 엔진(Physics-based) 느낌을 살립니다.

- **심야의 약방 (Floating)** [완료 ✅]: 메인 화면의 약사 캐릭터가 위아래로 미세하게 떠 있는 듯한 움직임 (y: [-10, 10], duration: 4s).
- **조제 중 (Bouncing)** [완료 ✅]: 고민 입력 후 약사 캐릭터가 약을 짓는 귀여운 바운스 애니메이션.
- **처방전 공개 (Slide-up & Fade)** [완료 ✅]: 카드가 화면 하단에서 슬라이드 업되며 서서히 선명해지는 효과(Staggered 애니메이션 포함).
- **스프링 탭 햅틱 (BounceTappable)** [완료 ✅]: 버튼/카드 터치 시 스프링처럼 반응하고 햅틱 피드백 제공.

## 4. 사용자 여정 상세 (User Flow) [완료 ✅]

### Step 1: 고민 접수 (Consultation) [완료 ✅]

- **UI**: 은은한 전구색 조명이 비치는 종이 질감 배경.
- **UX**: "오늘 어떤 마음을 처방해 드릴까요?" 문구와 함께 부드러운 텍스트 박스.

### Step 2: 스타일 선택 (Style Selection) [완료 ✅]

- **UI**: 3가지 테마(T/F/W)의 선택 폼.
- **디테일**: 선택 시 부드러운 그림자(Shadow) 효과 적용 및 약한 햅틱 피드백 (BounceTappable 연동).

### Step 3: 결제 및 조제 (Payment & Alchemy) [완료 ✅]

- **UI**: 딱딱한 스피너 대신 Bouncing 하트 애니메이션과 로딩 텍스트 노출 (SumpyoLoadingIndicator).

### Step 4: 처방 확인 (Prescription) [완료 ✅]

- **UI**: 메인 화면 폼과 처방 내역 페이지를 하단 탭 바(BottomNavigationBar)로 분리.
- **UX**: 화면 하단에서 올라오는 커스텀 모달 다이얼로그로 종이 질감의 처방전 확인.

## 5. 추가 진행해야 할 작업 (Next Phases) -> 신규 로드맵(Phase 4, 5)으로 이관 예정

- **마음 일기**: 지난 처방전들을 날짜별로 모아보는 '나만의 약장' 그리드 뷰 구현.
- **감정 분석 차트**: 사용자의 최근 감정 변화를 부드러운 꺾은선 그래프(Bézier curve)로 시각화.
- **성능 검증**: TickerProvider 최적화 및 RepaintBoundary 적용 확인.
